Parameters: - providers (required) — JSON string of [{slug, label}] for enabled providers (the model catalog itself is fetched dynamically on first popover open via ``automation:agent_models``) - field_name_model (default — name of the model input mirroring the selected "agent_model") ``provider:model`` spec. Rendered sr-only when ``required=True`` (so HTML5 validation fires on empty submit), otherwise ``type=hidden``. In ``seed_default=True`` mode the picker pre-selects ``default_model`` so the value is usually a concrete spec at submit time; in ``seed_default=False`` (settings) an empty submission is a meaningful "use the configured default" save. - field_name_thinking (default — name of the hidden input mirroring the chosen "agent_thinking_level") effort level; empty string = unset. - initial_agent_model (default "") — pre-selected spec, e.g. ``openrouter:anthropic/claude-haiku-4.5`` - initial_thinking_level (default "") — pre-selected effort: ``minimal|low|medium|high`` - initial_model_display (default "") — display-only form of ``initial_agent_model`` used in the locked pill (provider prefix and ``org/`` stripped). - default_model (default "") — full ``provider:model`` spec the picker pre-selects when no value is stored. In ``seed_default=True`` mode (the run-time pickers) the default also becomes the submitted value when the user doesn't pick. In ``seed_default=False`` mode (settings) it is shown as a hint only — the input stays empty so save persists "use the configured default". - default_thinking (default "") — effort level analogue of ``default_model``. - placeholder_label (default — pill label when the picker is unselected. Defaults "Pick a model") to the run-time prompt; settings pass a "Default (…)" hint derived from the env / Pydantic default. - with_effort (default True) — render the thinking-effort row inside the popover. Standalone model_name fields in settings pass False. - required (default True) — when True, the sr-only model input is ``required`` while empty, so HTML5 validation refuses submission with no model. Settings pass False because an empty value is a meaningful "use configured default" save. - seed_default (default True) — when True (run-time pickers), seed the picker with ``default_model`` / ``default_thinking`` so the submitted value is non-empty at first turn. When False (settings), don't seed — keep the input empty so the form can persist NULL ("use default"). - stale (default False) — when True, the pre-selected model references a provider that is no longer enabled. Renders a muted hint so users notice the value is dead. - disabled (default False) — render a static, locked pill (no popover) when the value cannot be changed (e.g. mid-conversation, where the backend ignores agent overrides after the first turn). - locked_title (default — tooltip used on the locked pill when ``disabled=True`` "Locked for this conversation") and the pill is not ``stale``. Run-time callers (chat, activity) leave the default; settings env-locked fields pass "Locked by environment variable" so the tooltip reflects the actual reason the field is read-only. {% comment %} The model input is sr-only rather than ``type="hidden"`` because the HTML5 constraint-validation spec bars hidden inputs from validation — ``required`` on a hidden input is silently ignored. The sr-only pattern keeps the field out of the visual layout while leaving it in the validation flow so the browser refuses to submit an empty selection. Settings pickers pass ``required=False`` because an empty submission is a meaningful save state ("use the configured default"), so the input is rendered ``type="hidden"`` instead — keeping it out of the validation flow entirely. {% endcomment %} {% if with_effort|default_if_none:True %} {% endif %}
